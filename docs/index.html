pgGit: Version Control for PostgreSQL Schemas

Branch, merge, and track your database changes like you do with code.

Get Started Quick Start

The Problem

Have you ever stared at a database after a late-night deployment wondering what changed? Lost track of which migration ran where? Wished you could just git checkout your database schema?

The Solution

pgGit brings Git-like version control directly into PostgreSQL. No external tools, no migration files to manage, just pure SQL functions that track every schema change automatically.

Quick Start

-- Install the extension
CREATE EXTENSION pggit;

-- Check your current status
SELECT * FROM pggit.status();

-- Create a new feature branch
SELECT pggit.checkout_b('feature/new-tables');

-- Make your schema changes
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    email TEXT UNIQUE NOT NULL
);

-- Commit your changes
SELECT pggit.commit('Add users table');

-- Merge back to main
SELECT pggit.checkout('main');
SELECT pggit.merge('feature/new-tables');

Key Features

Automatic Tracking

Every DDL operation is automatically captured. No manual migration files.

Branching & Merging

Create feature branches, work in isolation, merge when ready.

Time Travel

View your schema at any point in history. Rollback when needed.

Conflict Detection

Smart three-way merge with automatic conflict detection.

Pure PostgreSQL

No external dependencies. Just install and use.
